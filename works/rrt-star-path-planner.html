RRT* Robot Path Planner 🧭

  • Source Available Upon Request
  • Feb 2026
  • Robotics + Algorithms

Overview

The RRT* Robot Path Planner project focused on helping a robot find a safe path through maze-like environments. I implemented a sampling-based path planning algorithm that builds a tree of possible robot movements while avoiding obstacles in the environment.

Unlike a basic path search on a fixed grid, RRT* explores continuous space by randomly sampling points, connecting valid movements, and improving the path over time through rewiring. This made the project a strong introduction to real robotics planning problems where the robot has to move through space safely and efficiently.

The goal was to generate a collision-free path from a starting position to a goal position, then improve that path so the robot could follow a smoother and more practical route.

What I Built

I implemented the core logic for an RRT* path planner, including random sampling, nearest-neighbor search, steering toward sampled points, collision checking, cost tracking, and rewiring nearby nodes to improve path quality.

I also worked on path smoothing so the final path was not just valid, but easier for a robot to follow. This helped connect the algorithmic side of the project with the real movement constraints of a simulated robot.

Key Features

• RRT* sampling-based path planning algorithm
• Random point sampling in maze environments
• Nearest-neighbor search for expanding the planning tree
• Collision checking to avoid invalid robot paths
• Cost tracking and node rewiring to improve path quality
• Path smoothing for cleaner robot navigation
• Robotics simulation setup for testing planned paths

Impact

This project helped me better understand how robots make movement decisions in uncertain or obstacle-filled spaces. It also gave me hands-on experience with planning algorithms, geometry, collision detection, and the difference between a path that works in theory and a path that is useful for an actual robot.

It strengthened my ability to translate an algorithm from robotics theory into working code that can be tested visually in a simulated environment.
