RGB-D Tabletop Instance Segmentation 📷

  • Source Available Upon Request
  • Jan 2026
  • Robotics + Computer Vision

Overview

The RGB-D Tabletop Instance Segmentation project focused on helping a robot identify separate objects on a tabletop using depth and color data. Instead of only working with a flat image, the project used RGB-D information to reason about both appearance and 3D structure.

The main goal was to separate tabletop objects from the table surface, then group the remaining points into individual object clusters. This type of perception pipeline is important for robotics tasks where a robot needs to understand what objects are in front of it before planning how to interact with them.

This project gave me hands-on experience with computer vision, 3D perception, point cloud processing, and robotics-style object segmentation.

What I Built

I built a perception pipeline that processed RGB-D data, removed the tabletop plane, and segmented visible objects into separate instances. The project used RANSAC plane fitting to estimate and remove the table surface, then applied 3D clustering to group object points together.

I also worked through debugging segmentation results by checking how parameters affected plane detection, clustering quality, and whether objects were correctly separated from the background.

Key Features

• RGB-D data processing for color and depth-based perception
• RANSAC plane fitting to detect and remove the tabletop surface
• Point cloud filtering to isolate object points from background points
• 3D clustering to separate individual tabletop objects
• Instance segmentation pipeline for robotics perception tasks
• Parameter tuning for improving object separation and segmentation quality

Impact

This project helped me understand how robots interpret the physical world through sensors. It showed me that perception is not just about detecting pixels, but about turning noisy sensor data into useful structure that a robot can act on.

It also strengthened my ability to reason about 3D data, tune algorithms, and debug visual outputs when the segmentation results did not match the real scene.
