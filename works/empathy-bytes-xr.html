Overview

Empathy Bytes is an educational archival platform focused on using technology to preserve, share, and explore cultural and historical experiences. My work on the project involved improving the web experience, supporting frontend development, and contributing to interactive XR-related features.

The platform combines web development, content management, UI design, and immersive technology. I worked with a React-based Gatsby frontend, Drupal-backed content, JavaScript features, and design improvements that helped make the site easier to navigate and maintain.

This project gave me experience working on a real team-based platform where technical decisions had to support users, content editors, researchers, and future contributors.

What I Built

I contributed full-stack and frontend features for the Empathy Bytes platform, including interface updates, page redesign work, styling improvements, and content-connected components. I also worked on improving the development workflow by testing changes locally and debugging differences between the local and live site.

Beyond the standard website work, I explored XR and 3D-related features using tools like Unity and Luma AI. This helped connect the platform’s educational mission with more interactive and immersive experiences.

Key Features

• React/Gatsby frontend updates for an educational archive platform
• Drupal-connected content structure for easier site maintenance
• Redesigned article and media page experiences
• Responsive styling improvements based on Figma designs
• Debugging for local versus live deployment differences
• XR and 3D exploration using Unity and Luma AI
• Team-based development across design, content, and engineering needs

Impact

This project strengthened my experience building technology for real users instead of isolated assignments. I had to think about maintainability, design accuracy, accessibility, content workflows, and how technical choices affect the people updating and using the platform.

It also helped me grow as a frontend developer because I worked through real deployment and styling issues, not just local code. I learned how important it is to build features that look good, work consistently, and remain easy for a team to update over time.
