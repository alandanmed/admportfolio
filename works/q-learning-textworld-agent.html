Q-Learning TextWorld Agent 🧠

  • Source Available Upon Request
  • Aug 2025
  • Reinforcement Learning

Overview

The Q-Learning TextWorld Agent project focused on training an agent to make decisions in a text-based environment using reinforcement learning. Instead of being directly told the correct action at every step, the agent learned through trial, error, rewards, and repeated interaction with the environment.

The project used Q-learning to help the agent estimate which actions were valuable in different states. Over time, the agent updated its Q-values based on the rewards it received and the future value of the states it reached.

This project helped me understand how reinforcement learning agents learn policies from experience, especially in environments where the agent has to balance exploration and exploitation.

What I Built

I implemented the core logic for a tabular Q-learning agent, including state representation, action selection, Q-value updates, reward tracking, and policy execution.

I also worked with exploration strategies so the agent could try new actions early in training while gradually relying more on learned behavior. This helped connect the theory of reinforcement learning with actual agent performance inside a TextWorld environment.

Key Features

• Tabular Q-learning implementation for decision making
• State-action value tracking through Q-values
• Exploration and exploitation balance during training
• Reward-based learning from environment feedback
• Policy execution based on learned Q-values
• TextWorld environment interaction for text-based tasks
• Training loop for repeated agent improvement over time

Impact

This project strengthened my understanding of reinforcement learning fundamentals. It helped me see how an agent can improve its decisions without supervised labels, using only rewards and experience from the environment.

It also gave me more practice debugging learning behavior. When an agent performs poorly, the issue is not always obvious. It could come from the reward structure, state representation, exploration rate, update rule, or training duration.
